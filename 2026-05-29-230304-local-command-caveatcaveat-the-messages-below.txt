❯ # 다크/라이트 모드 토글 재추가 작업                                           
  # 대상: app.py + 화면0~4 코드(필요 시 각 화면 모듈 파일)                      
  # 작업 전 app.py 를 app.py.bak_theme 로 백업하라.                             
                                                                                
  ## 0. 사전 진단                                                               
  1. 과거에 다크/라이트 토글 코드가 있었던 흔적이 남아 있는지 확인하라          
     (주석 처리된 코드·미사용 함수·st.session_state.theme 등).                  
     잔재가 있으면 위치를 보고하라(이번 작업에서 통일된 구현으로 교체한다).     
  2. 화면4 원본 HTML(screen4_template.html)과 화면0 HTML(screen0.html)에        
     자체 테마 버튼(🌙/☀️ 등)이 들어 있는지 확인하라.                           
     들어 있으면 위치를 보고하라(자체 버튼은 비활성화/제거하고                  
     Streamlit 토글로 통합한다).                                                
  3. 화면별로 적용 중인 CSS 클래스(.screen3-wrap, .screen4-wrap 등)를           
     목록으로 보고하라.                                                         
                                                                                
  ## 1. 전역 테마 상태 관리                                                     
  - st.session_state.theme 을 도입한다. 값은 "dark" 또는 "light".               
  - 앱 시작 시 기본값은 "dark" 로 초기화하라.                                   
  - 사이드바(st.sidebar)에 토글 위젯을 둔다:                                    
      st.sidebar.toggle("🌙 다크 모드", value=(st.session_state.theme=="dark"), 
                        key="theme_toggle", on_change=...)                      
    on_change 콜백에서 theme 값을 갱신하고 st.rerun() 한다.                     
  - 토글 위치를 변경하고 싶으면 추후 헤더로 옮길 수 있도록 위젯을 함수로        
  분리하라.                                                                     
    예: render_theme_toggle() — 호출 위치만 바꾸면 되게.                        
                                                                                
  ## 2. 공통 테마 적용 함수                                                     
  - 공통 함수 apply_theme() 을 정의하라(파일 위치: app.py 상단 또는 theme.py    
  신설).                                                                        
  - 함수는 st.session_state.theme 값에 따라 아래 CSS 를 st.markdown 으로        
  주입한다.                                                                     
  - CSS 는 다크/라이트 양쪽에 대해 동일한 변수명(--bg, --card, --card2,         
  --border,                                                                     
    --text, --text2, --text3, --accent, --blue, --green, --red, --purple,       
    --shadow, --radius, --radius-sm 등)을 정의하라.                             
  - 다크/라이트 값은 원본 화면4 HTML(screen4_template.html) 안의                
    :root / body.light 정의를 참고해 동일한 팔레트로 통일하라.                  
  - 변수 적용 스코프:                                                           
    · 가능하면 body 또는 .stApp 루트에 적용해 모든 화면에 자동 반영되게 한다.   
    · 단 화면별 .screenN-wrap 스코프가 이미 있는 곳은 그 스코프에서도           
      같은 변수를 그대로 상속하도록 작성하라.                                   
  - 토글 상태 표시: 다크일 때는 "🌙 다크 모드", 라이트일 때는 "☀️ 라이트 모드"  
    로 라벨이 바뀌게 하라.                                                      
                                                                                
  ## 3. 모든 화면에서 일관 적용                                                 
  - app.py 의 page 분기 "최상단"에서 매 rerun마다 apply_theme() 을 1회          
  호출하라.                                                                     
  - 화면0~4 의 어느 화면에 들어가도 동일한 테마가 적용되어야 한다.              
  - 화면별 모듈(screen3_plan.py, screen4_career.py 등)이 자체적으로 CSS 를      
    주입하고 있다면, 그 CSS 의 색상 하드코딩을 제거하고 위 공통 변수            
    (var(--bg), var(--text) 등)를 사용하도록 바꿔라.                            
                                                                                
  ## 4. HTML 템플릿 충돌 방지                                                   
  - screen0.html, screen4_template.html 등에 자체 다크/라이트 토글 버튼이       
    들어 있으면 비활성화하거나 제거하라(중복 토글 방지).                        
  - HTML 내부 CSS 가 자체 :root / body.light 를 정의하는 경우,                  
    Streamlit 측 변수와 충돌하지 않게 하라:                                     
    · 가장 간단한 방법: HTML 내부 :root / body.light 정의를 그대로 두고,        
      Streamlit 에서 body 에 class="light" 또는 data-theme 속성을               
      apply_theme() 시점에 부여하는 JS 한 줄 주입.                              
    · 또는 HTML 내부 색상 정의를 제거하고 외부 변수를 상속하도록 통일.          
    둘 중 안정적인 방법을 선택해 적용하라.
